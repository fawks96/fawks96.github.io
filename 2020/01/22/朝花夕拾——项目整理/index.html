朝花夕拾——项目整理

截止现在（2020年寒假），我的研究生生涯刚好过半了。回望过去，才发现这一年多里，竟都没时间完成一件之前计划了很久的事，那就是把自大学以来的一些项目进行整理和回顾。形式上，这种整理应当是对项目代码的系统整理和优化。更深层次的，希望能通过这样的整理对之前的编程经历（设计逻辑、实现思路、效果展现等方向）有一个思想层次上的整理和提升。

然计划总赶不上变化。

一方面，学业上的重担一直未曾减轻。研一一整年的精力投入繁忙的课业学习中（除了主修的专业必修大数据课程及其实践，还有若干深度学习、数据可视化、数据分析等若干项目需要完成，以及一些英语和自然辩证法等文科类课程）。不仅如此，导师平时还不时有相关的调研任务下发，如研一上由于导师想搞一台超算来为项目提供计算力支持，一个学期都在不停的对超算相关的知识进行学习和调研，这也占用了相当的课余时间。然后本以为到了研二会轻松一些，然而又要开始做数据挖掘相关的实验，同时在写两篇论文。好在结果差强人意，学期结束后两篇文章一篇已投出并录用，另一篇也完成了并投出。理想的话研二这一年我的成果应该就有两篇EI文章了。这应该也就是我短暂的所谓科研生涯的成果了吧。在这两年时间里，对科研圈也算进行了比较深入的了解了，发现确实不是自己应该深入踏入的领域，这匆匆的一瞥对我来说已经够了，剩下来的时间应该投入更多实践的领域了，我想这也是我此番朝花夕拾的起因之一吧。

另一方面，由于时间过的比较久了（说久也不久，说短也不短，大学时候的一些项目离现在一般都有个两三年的时间了），很多项目甚至都不记得当初的很多思路和想法了，以至于要以一个近乎新人的角度重新进行理解当年的项目。除了时间的因素外，项目的质量也没有特别高的保证。这有课后项目项目提交时间紧，导致项目没有很高的完成度的原因，当然更有自身水平有限的原因。所以心底其实是有点保留甚至畏惧进行这样的回顾的，因为一方面需要耗费较多的时间精力，另一方面最终的结果可能不会特别的好，毕竟学生时代的项目总归没有很高的质量保证。

不管怎样，还是希望能对过往进行回望，以便更好的出发。

以下是我进行整理的项目列表，会一直进行更新，直到完成整理为止。

SQL Injection Attack Project

项目地址

这与其说是个项目，不如说是一段实践更为恰当。我几乎都快遗忘了它的存在，直到我在Google Drive中翻阅本科时候的一些资料的时候，才发现了当年自己的这一段黑客经历。

起因是当时在上数据库课，然后在读课本的时候发现了有一个小节是讲SQL注入的，篇幅不长，但居然引起了我的兴趣。平时自己也有逛乌云网站（后来这个网站彻底凉了。。），从上面看到了很多“白客”如何挖掘各种网站的安全漏洞。出于好奇的目的，我在乌云上搜索了自己学校，发现居然有好几个针对本校网站的漏洞信息，其实大多数都是利用的SQL注入。为了一探究竟，一场探索之旅就此展开…

利用的工具是SQL注入的利器SQLMAP工具，目标是学校的官方OJ平台（acm.cs.ecnu.edu.cn），最终的成果是获取了网站的后台最高管理权限。并将此漏洞对学院的老师进行了上报。
